# AGENTS.md

Быстрый вход для ИИ-агентов, которые будут работать с репозиторием дальше.

## Что это за проект

`soundcloud_bot` — Telegram-бот для скачивания аудио и видео по ссылкам из SoundCloud и YouTube. Текущая модель продукта:

- пользовательский сценарий бесплатный для всех;
- месячных лимитов запросов нет;
- единый максимум длительности медиа — 3 часа;
- metadata-редактирование `mp3` доступно всем;
- монетизация идет через стороннюю рекламу;
- админский контур содержит RBAC, аудит, рассылки и управление рекламными слотами;
- продовый деплой идет через Docker Compose.

## Что считать источником истины

- По фактическому поведению продукта сначала доверять коду.
- `DEVELOPER_GUIDE.md` и `RUNBOOK.md` полезны, но написаны под человека-оператора.
- Payment/Robokassa/Stars-модули и SQL-миграции могут оставаться как legacy-след, но активный runtime их не импортирует и не запускает.

## Критические инварианты

- Пользователю нельзя показывать Premium, подписки, оплату, тарифы или месячные лимиты.
- Количество запросов не ограничивается месячным usage-счетчиком.
- Максимальная длительность любого медиа — 3 часа.
- Рекламные сообщения должны формироваться кодом и содержать маркировку `Реклама`, рекламодателя и `erid`.
- Нельзя логировать сырые токены, DSN, webhook-body, invoice payload, рекламные договорные секреты и другие чувствительные данные.
- `docker-compose.yml` и `Dockerfile` содержат меры hardening; не ослаблять их без явного запроса.
- `cookies.txt` — чувствительный артефакт. Не копировать его в новые места, не логировать содержимое, не монтировать на запись.
- Metadata-сессии и временные файлы должны жить временно и очищаться.
- Любое расширение хранения персональных данных или рекламной аналитики должно сопровождаться обновлением `docs/AI_SYSTEM_MAP.md`, `docs/LEGAL_STATUS.md` и legal-шаблонов.

## Особо чувствительные зоны

- `app/handlers/downloads.py` — основной пользовательский сценарий, duration-check, выдача файла и metadata prompt.
- `app/ads_store.py` — рекламные кампании и статистика ручных рекламных рассылок.
- `app/services/worker.py` — fallback-загрузка файлов на внешние сервисы `gofile/file.io`, если Telegram не принял файл.
- `app/access.py` — роли, admin nonce, аудит и bootstrap superadmin.
- `app/logging_utils.py` — санитарная обработка логов.

## Что хранится

- Redis или локальный fallback: настройки, профили, dedup, admin nonce, metadata-сессии, рекламные кампании и статистика показов.
- Файловая система: временные медиафайлы, metadata-рабочие копии, логи, `cookies.txt`.
- Legacy PostgreSQL-схема платежей может существовать в старых окружениях, но новый runtime от нее не зависит.

## Проверка после изменений

```powershell
python -m pytest -q -p no:cacheprovider
```

Если меняли рекламу или пользовательский download-flow, отдельно проверить:

```powershell
python -m pytest -q -p no:cacheprovider tests\test_ads_policy_runtime.py tests\test_downloads_handlers.py
```

## Когда нужно обязательно обновлять документацию

- изменился рекламный формат, режим запуска или состав рекламных данных;
- изменился состав пользовательских данных;
- добавились новые внешние интеграции;
- изменились retention/TTL/хранилища;
- изменился UX акцепта оферты или согласия на персональные данные;
- изменился продовый деплой.
